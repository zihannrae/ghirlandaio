## Pendahuluan
perangkat lunak (software), telah menjadi bagian penting dari kehidupan sehari-hari di era digital saat ini, termasuk aplikasi produktivitas dan platform media sosial. Akan tetapi, pada kenyataannya, tidak semua software memberi pengguna akses yang sama. Ada dua model software: sumber terbuka (kode sumber terbuka) dan sumber tertutup (kode sumber tertutup). Dalam sistem sumber tertutup, pengembang atau perusahaan pemilik menyembunyikan kode, sehingga hanya mereka yang dapat mengubah, memperbarui, atau menyebarluaskan perangkat tersebut. Tujuannya adalah untuk memastikan bahwa perangkat lunak yang digunakan sesuai dengan lisensi dan peraturan pemiliknya.
Contoh perangkat lunak closed sources masih mendominasi pasaran, seperti Microsoft Windows, Adobe Photoshop dan aplikas IOS.  Meskipun menawarkan kemudahan dan dukungan resmi dari si pemilik, model perangkat lunak ini membawa berbagai dampak bagi penggunanya. 

## Pembahasan
Closed sources atau kode sumber tertutup adalah model pengembangan perangkat lunak dimana kode sumbernya tidak dapat diakses atau dimodifikasi secara umum selain pemiliknya. Salah satu contoh perangkat lunak closed sources yang terkenal Adalah sistem operasinya Windows dari Microsof. Windows dikembangkan oleh Microsoft dan kode sumbernya tidak dapat diakses oleh public, sehingga hanya pihak Microsoft yang memiliki control penuh atas perangkat tersebut. Karena kode sumbernya yang diciptakan tertutup untuk public maka, perangkat ini lebih sulit bagi penyerang untuk menemukan dan mengeksploitasi kerentanan keamanan. 
Biarpun begitu, di salah satu situs menyebutkan jika  closed sources juga memiliki dampak bagi pengguna, yaitu jika kode sumbernya yang tidak dapat diakses atau  dimodifikasi oleh pengguna. Ya saya setuju dengan pernyataan ini sehingga mengakibatkan  pengguna tidak dapat memiliki control penuh atas perangkat lunak tersebut serta tidak dapat mengkutomisasi sesuai kebutuhan pengguna. Sehingga fleksibilitas sangat terbatas, membatasi untuk menciptakan inovasi dan ketidak adanya efisien, terutama bagi pengguna dengan kebutuhan unik, jika sistem atau 
aplikasi mengalami bug harus menunggu updatean dari si pemilik. Secara keseluruhanadanya pembatasan akses kode membuat pengguna kehilangan transparansi dan kendali.

## Kesimpulan
Pembatasan akses kode pada sistem close souce secara signifikan memengaruhi pengguna dengan fleksibilitas dan kendali atas software yang pengguna gunakan sehari – hari. Meskipun model ini menawarkan kemudahan penggunaan, dampak negative  bagi pengguna tetap dominan.
